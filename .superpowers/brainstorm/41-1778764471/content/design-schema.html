Design: Claude Sonnet Extraction Schema

What the AI extracts from each incident. Old schema (left) vs proposed new schema (right).

Current Schema — GPT-4o (4 fields)
"explicit_cause_reported": "string"
"latent_hardware_software_issues": ["string"]
"latent_human_environmental_factors": ["string"]
"pattern_discovery_summary": "string"
✗ No structured categories — free text only
✗ Themes/clustering entirely dependent on embedding similarity
✗ Nothing usable directly as a dashboard filter
✗ No severity or consequence weighting
✗ No regulatory context
New Schema — Claude Sonnet 4.6 (structured)
CLASSIFICATION
"incident_category": "navigation|mooring|crew_injury|machinery|fire|flooding|contact|other"
"vessel_activity": "underway|berthing|anchored|cargo_ops|maintenance|other"
"immediate_cause": "string"

CONTRIBUTING FACTORS
"contributing_factors": [{
  "type": "human|hardware|environmental|procedural",
  "description": "string",
  "confidence": "high|medium|low"
}]

HUMAN FACTORS
"fatigue_factor": true|false|null
"training_factor": true|false|null
"communication_factor": true|false|null
"ppe_factor": true|false|null

ENVIRONMENT LINKAGE
"weather_was_factor": true|false|null
"lighting_was_factor": true|false|null

REGULATORY
"solas_chapters": ["II-1", "V"]
"preventable_by": ["string"]

PATTERN (kept for embedding)
"pattern_discovery_summary": "string"
✓ Every field is a direct dashboard filter
✓ Fatigue/training/PPE become quantified across all 7k incidents
✓ SOLAS chapter breakdown for regulatory audience
✓ Contributing factor types power the Themes page drill-down
✓ Pattern summary still feeds embedding + HDBSCAN

  • "X% of night incidents involved fatigue"
  • Fatigue factor trend by year
  • PPE deficiency rate over time

Themes page
  • SOLAS chapter breakdown per theme
  • Top contributing factor types
